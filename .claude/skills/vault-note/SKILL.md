# Vault Note — `/vault-note SLUG`

Writes (or refreshes) the **`note`** prose on **one** vault page in
`ingest/store/vault.json`. The note is the only human/Claude-owned field on a
page; everything under `auto.*` (refs, stats) is machine-generated by
`vault_sync.py` and **must never be touched here**. Runs entirely in a Claude
Code session — no API key needed.

## What a vault note is

- **Plain prose, 1–3 tight paragraphs.** Not a memo (those live in
  `memos.json` with fixed sections) — this is the connective-tissue
  description of what a page *is* and how it relates to the rest of the vault.
- **Dense with `[[wikilinks]]`** to related pages — that density is the point:
  every wikilink becomes a graph edge and a backlink. Link to member/related
  tickers, the parent/adjacent themes, the concepts it depends on, and
  `[[aleabitoreddit]]` where his stance is the subject.
- Same inline markup as memos: `**bold**`, `$TICK`, `[[wikilink]]` only.

## Procedure

1. **Find the page** in `ingest/store/vault.json` by `slug` (kebab-case, e.g.
   `photonics`, `aaoi`, `hbm`, `aleabitoreddit`). If it doesn't exist, stop:
   sync creates pages from the store — the name may be radar-tier (no page) or
   need `vault_sync.py` run first.

2. **Gather grounding** (read-only), scoped to the page type:
   - **theme**: its Brain digest in `brain.json` (narrative + key points), its
     member tickers (`auto.refs`), `base.json` category metadata
     (subtitle, tooltip, investorAngle).
   - **ticker**: its theses in `theses.json`, its desk verdict in
     `verdicts.json`, its record in `tickers.json`, its category's digest.
   - **concept**: its glossary `def` in `base.json`, and the theses/themes
     where the term actually bites.
   - **person**: the digests and verdicts in aggregate — whose views these are.

3. **Verify every `[[wikilink]]` target resolves** to an existing page slug
   before writing (a dangling link renders but adds no edge and dead-ends).
   Ticker slugs are the lowercased symbol; `.`→`-` (e.g. `LPK.DE` → `lpk-de`).
   Where a slug reads badly as link text, lead with the readable name:
   `LPKF ([[lpk-de]])`.

4. **Write only `note` + `noteUpdatedAt`** on that one page object. Set
   `noteUpdatedAt` to today (ISO, e.g. `2026-07-12T00:00:00Z`). Leave
   `slug`, `type`, `title`, and the entire `auto` block untouched.

5. **Regenerate + verify**:
   - `python3 ingest/generate_data_js.py` — this re-runs `vault_sync.sync()`,
     which **preserves your note** and recomputes `auto`/`meta` (pageCount,
     linkCount) around it.
   - `python3 -m unittest discover -s ingest/tests`
   - Open `vault.html?page=<SLUG>` — the note renders, wikilinks are clickable,
     "What links here" and the graph pick up the new edges; no console errors.

6. **Report**: one line on what the note now says and which pages it links to.

## Hard rules

- **Never modify `auto.*`** — it's owned by `vault_sync.py`. Notes in, stats out.
- Never fabricate a number. Analyst-stated targets quoted from theses are fine
  (they're claims); live figures (price, tier, mentions) render from `auto`.
- Never hand-edit `data.js` (CLAUDE.md rule #6); regenerate it.
- Research support, not investment advice.
